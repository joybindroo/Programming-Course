# Distributed Systems

## Overview

This module introduces the fundamentals of distributed systems, covering concepts such as:

- **Scalability** – horizontal vs. vertical scaling
- **Reliability** – fault tolerance, replication, consensus algorithms
- **Consistency Models** – strong, eventual, CAP theorem
- **Communication** – RPC, messaging queues, gRPC, REST
- **Coordination Services** – Zookeeper, etcd, Consul
- **Microservices Architecture** – service discovery, API gateways
- **Distributed Data Stores** – NoSQL, distributed SQL, sharding
- **Security** – authentication, encryption, zero‑trust networking

### Learning Objectives

1. Explain the key challenges in building distributed systems.
2. Compare different consistency models and their trade‑offs.
3. Design a simple microservice‑based application using Docker and a coordination service.
4. Implement a basic client‑server communication pattern using gRPC.

### Suggested Exercises

- Set up a two‑node etcd cluster and store a key‑value pair.
- Build a simple chat application using WebSockets across multiple containers.
- Analyze the CAP theorem with real‑world examples (e.g., Cassandra vs. PostgreSQL).
