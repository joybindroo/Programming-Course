# Module 17 – Capstone Project Management

---

## Learning Objectives
- Plan and scope a multi‑module software project using Agile/Scrum.
- Define milestones, deliverables, and success criteria.
- Manage stakeholder communication, risk registers, and change requests.
- Produce essential documentation: project charter, backlog, sprint plans, demo video, post‑mortem.
- Reflect on lessons learned and create a personal development roadmap.

## Overview
All prior modules build technical skills; this final module ties them together in a real‑world capstone. You will orchestrate a full‑stack application, from requirements to deployment, applying best‑practice project management.

## Key Topics
1. **Agile Foundations** – Scrum roles, ceremonies, artifacts.
2. **Project Charter** – vision, scope, constraints, assumptions.
3. **Backlog Grooming** – user stories, acceptance criteria, story points.
4. **Sprint Planning & Review** – task breakdown, demo preparation.
5. **Risk Management** – identification, mitigation, contingency.
6. **Documentation** – architecture diagram, API spec, README, changelog.
7. **Demo & Presentation** – recording a walkthrough, stakeholder pitch.
8. **Post‑mortem & Retrospective** – what went well, improvements, future work.

## Mini‑Lab
1. Choose a **capstone idea** that integrates at least 5 prior modules (e.g., a web app with Docker, CI/CD, database, AI feature, and testing).
2. Draft a **project charter** and create a **GitHub project board** with epics and tasks.
3. Conduct **two sprint cycles** (1‑week each), delivering incremental functionality.
4. Record a **demo video** (screen capture + narration) and write a **post‑mortem report**.
5. Publish the final repository with a polished README, CI pipeline, and deployment instructions.
